## Why Newest First Matters

### Scenario: Database Grows to 15,000 Records

#### With Oldest First (Bad)
```
Database: 15,000 records
Limit: 10,000
Fetched: Rows 1-10,000 (oldest)
Missing: Rows 10,001-15,000 (newest) âŒ

User sees: Old data
User doesn't see: Their recent work âŒ
```

#### With Newest First (Good)
```
Database: 15,000 records
Limit: 10,000
Fetched: Rows 15,000-5,001 (newest)
Missing: Rows 1-5,000 (oldest) âœ…

User sees: Recent data âœ…
User doesn't see: Very old data (acceptable)
```

**Conclusion:** Even if we hit the limit again, users will see their most recent work.
